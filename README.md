# Study

这是一个通用教程工程仓库，用来系统整理不同技术主题的学习路径、章节设计和成体系的教学内容。

它的目标不是零散堆资料，而是把一个主题真正做成“可以从头读到尾”的教程项目。

## 项目定位

这个仓库关注的是教程如何被设计出来，而不只是教程内容本身。

核心问题包括：

- 一个复杂主题应该怎样拆章节
- 章节为什么按这个顺序组织
- 如何让读者从不会到会
- 如何让不同主题复用同一套教程工程结构

## 当前内容

- `prd.md`：仓库级通用教程总 PRD，定义教程工程的分层结构和教学设计方法
- `spec/`：围绕需求、规格与软件工程文档体系的完整教程主题
- `window_functions/`：窗函数学习与可视化资料，包含常见窗函数的时域形状和频域幅度响应 JPG 图

`spec/` 当前已经包含从概念入门到完整案例的连续章节，覆盖：

- 什么是需求与规格
- 规格在软件流程中的位置
- PRD、User Story、Acceptance Criteria、功能 Spec 的写法
- 业务规则、状态、数据边界、非功能需求
- 规格评审、常见错误、案例实战

`window_functions/` 当前包含一个可重复运行的绘图脚本和 20 类常见窗函数图像。每张图同时展示窗函数在时域中的权重形状，以及它在频域中的主瓣、旁瓣和衰减特征，适合用于理解频谱分析、FFT 泄漏控制、FIR 滤波器设计和信号截断带来的影响。

## 设计思路

这个仓库强调三件事：

1. 教程不是文章集合，而是学习路径
2. 每一章都必须有明确教学目的
3. 同一套结构应当可以复用到不同主题

因此仓库默认会区分几层内容：

- 根目录总 `prd.md`
- 主题目录 `prd.md`
- 主题目录的阅读顺序文件
- 各章节正文

## 适用场景

这个仓库适合：

- 想把某个技术主题写成系统教程的人
- 想做长期维护型知识项目的人
- 希望教程既能顺序学习，也能按问题跳读的人

## 阅读建议

如果你第一次进入这个仓库，建议这样看：

1. 先读根目录 `prd.md`，理解教程工程的总设计
2. 进入 `spec/`，查看 `00 总目录与阅读顺序.md`
3. 再按章节顺序阅读具体内容

如果你想学习信号处理中的窗函数，可以进入 `window_functions/`，先阅读其中的 `README.md`，再对照每张 JPG 图片观察不同窗函数在时域和频域之间的取舍。

## 仓库定位总结

`Study` 更像一个“教程工程母版 + 主题化教学内容”的组合仓库。

它不仅记录知识点，更强调：

- 教学结构
- 章节职责
- 学习路径设计
- 长期可扩展的教程组织方式
